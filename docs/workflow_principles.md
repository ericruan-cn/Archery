# 工单规范

## 1. 规范背景与目的

为保障公司业务系统稳定、健康地进行升级，推出了本SQL审核平台。
审核平台为升级上线和业务数据调整提供了方便高效的操作和管理页面，极大的促进了数据库变更和处理流程的规范化、平台化、持久化。
SQL审核平台保留了SQL上线以来的所有历史和细节，便于概览和跟踪溯源。

## 2. 工单类型划分

2.1. `上线单`, 代表待进行系统上线的工单
2.2. `运维单`, 代表待进行数据调整的工单

## 3. 工单起名规范

3.1. 【强制】`上线单`，起名格式为：工单类型-系统名称-`子系统名称[或业务模块]`-`内容简述`
3.2. 【强制】`运维单`，起名格式为：工单类型-系统名称-`子系统名称[或业务模块]`-`内容简述`
3.3. 【优化】工单名称中工单类型和系统名称均为选择项，后面的子系统名称[或业务模块]和内容简述为手工输入
3.4. 【强制】工单起名示例

     上线单-环球云-[商品|订单|售后|财务|节目|合同|SCM|CRM|TMS|WMS]系统-简要描述 
     运维单-环球云-[商品|订单|售后|财务|节目|合同|SCM|CRM|TMS|WMS]系统-简要描述 
     
     上线单-云商城-[商品|订单|支付]系统-简要描述 
     运维单-云商城-[商品|订单|支付]系统-简要描述

## 4. `强制`

4.1 【`强制`】`ORACLE`的上线单如果会导致`数据库对象失效`(如`视图`、`函数`、`过程`、`包`、`触发器`等)，应在工单末尾补充以下内容：

      /** 重新编译失效对象 */
      call sp_recompile_invalid_object();

      /** 重新创建失效同义词 */
      call sp_recreate_invalid_synonym();
